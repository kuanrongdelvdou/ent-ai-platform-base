# 项目初始化

## 目标

搭建前后端项目骨架，确保本地开发环境可以正常运行。

## 目录结构

```
ent_ai_platform/
├── ai-admin/
│   ├── backend/                ← NestJS 后端
│   └── frontend/               ← Soybean Admin (Vue 3)
├── docker-compose.yml      ← 本地开发环境
└── docs/                   ← 项目文档
```

## 后端初始化（NestJS）

- Node.js 版本：≥ 20
- 包管理器：npm
- 使用 NestJS CLI 初始化项目
- 配置 TypeScript strict 模式
- 集成 ESLint + Prettier

依赖：
- `@nestjs/config` — 环境变量管理
- `prisma` + `@prisma/client` + `@prisma/adapter-pg` — PostgreSQL ORM（Prisma 7）
- `@nestjs/jwt` + `@nestjs/passport` — 认证
- `ioredis` — Redis 客户端（可选，第一阶段暂不使用）
- `class-validator` + `class-transformer` — 参数校验

已实现模块：
- `auth` — 登录认证（`/auth/login`、`/auth/getUserInfo`）
- `route` — 动态路由（`/route/getUserRoutes`、`/route/isRouteExist`）
- `systemManage` — 系统管理（用户/角色/菜单 CRUD）

待实现模块：
- `dept` — 部门管理
- `operationLog` — 操作日志
- `dict` — 字典管理
- `config` — 系统参数

## 前端初始化（Soybean Admin）

- 基于 [Soybean Admin](https://github.com/soybeanjs/soybean-admin)
- Vue 3 + TypeScript + Vite
- UnoCSS
- Naive UI 组件库
- Pinia 状态管理
- Vue Router（动态路由模式 `VITE_AUTH_ROUTE_MODE=dynamic`）
- 请求基础路径通过 Vite proxy 转发到本地 NestJS（`/proxy-default/`）

## Docker 本地环境

`docker-compose.yml` 启动：
- PostgreSQL 15
- Redis 7（可选，验证码、限流、缓存、token 黑名单需要时启用）

## 环境变量

```env
# 后端 .env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/ent_ai_platform

JWT_SECRET=dev-only-change-me
JWT_EXPIRES_IN=7d

# 前端 .env.test
VITE_SERVICE_BASE_URL=http://localhost:3000
VITE_SERVICE_SUCCESS_CODE=200
VITE_AUTH_ROUTE_MODE=dynamic
VITE_ROUTE_HOME=home
```

## 启动命令

```bash
# 后端（ai-admin/backend/）
npm run start:dev        # 开发模式（热重载），端口 3000
npx ts-node prisma/seed.ts  # 重置种子数据

# 前端（ai-admin/frontend/）
npm run dev              # 开发模式，访问 http://localhost:9527
```

默认账号：`soybean` / `soybean123`
