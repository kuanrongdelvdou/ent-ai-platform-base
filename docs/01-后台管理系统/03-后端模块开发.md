# 后端模块开发

## 技术架构

```
NestJS
├── 模块化结构（每个业务一个 Module）
├── JWT 认证（Passport + JwtAuthGuard）
├── AppLoggerService — 结构化中文日志（INFO/WARN/DEBUG/ERROR）
├── 统一响应格式（{ code: 200, msg: "ok", data: T }）
└── 统一异常处理（AllExceptionsFilter）
```

## 已实现模块

### 1. 认证模块 `AuthModule`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/login` | 登录，返回 `{ token, refreshToken }` |
| GET | `/auth/getUserInfo` | Bearer → `{ userId, userName, roles[], buttons[] }` |
| POST | `/auth/refreshToken` | 刷新 token |

## 日志规范

使用 `AppLoggerService`（`src/common/logger/app-logger.service.ts`），日志使用中文：

| 级别 | 方法 | 使用场景 |
|------|------|---------|
| INFO | `log.info()` | 登录成功、CRUD 操作完成 |
| WARN | `log.warn()` | 登录失败、业务校验失败、4xx 错误 |
| DEBUG | `log.debug()` | 查询参数、分页信息 |
| ERROR | `log.error()` | 5xx 异常、未预期错误 |
