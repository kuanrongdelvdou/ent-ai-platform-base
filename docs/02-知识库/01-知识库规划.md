# 知识库规划

## 目标

企业文件可以上传、管理、语义检索、权限隔离，支持 ≥ 10,000 个文件。

## 核心引擎

**RAGFlow**（已本地部署）

- 文档上传、解析、向量化
- 语义检索
- 支持 PDF、Word、PPT、Excel、图片、纯文本

---

## 功能模块

### 1. 知识库管理
- 知识库 CRUD（一个知识库对应一个 RAGFlow Dataset）
- 知识库与部门/权限绑定（谁能看哪个知识库）
- 知识库状态（解析中/就绪/失败）

### 2. 文件管理
- 文件上传（支持批量）
- 文件列表（分页、搜索、过滤）
- 文件解析状态查看
- 文件删除

### 3. 知识库检索
- 关键词检索
- 语义检索（向量相似度）
- 检索结果展示（命中段落高亮）

### 4. 权限隔离
- 知识库级别权限（角色可见性）
- 文件级别权限（可选，Phase 2 评估）

---

## 架构

```
前端（Soybean Admin / Vue 3）
  ↓
NestJS（封装 RAGFlow API，鉴权 + 限流 + 日志）
  ↓
RAGFlow REST API
  ↓
向量库（RAGFlow 内置 Milvus/Qdrant）
```

说明：

- RAGFlow 作为底层 RAG 引擎，负责知识库、文档解析、切片、向量化、检索和问答能力。
- 自建平台作为统一入口，负责登录、组织、用户、RBAC、资源授权、审计和前端体验。
- 前端不直接调用 RAGFlow，不暴露 RAGFlow API Key；所有请求先进入 NestJS，再由 NestJS 调用 RAGFlow API。
- 当前阶段优先走 API 对接，不使用 iframe 作为正式功能入口。

---

## NestJS 封装的 RAGFlow API

- 创建/删除知识库 → RAGFlow Dataset API
- 上传文件 → RAGFlow Document Upload API
- 查询解析状态 → RAGFlow Document Status API
- 语义检索 → RAGFlow Retrieval API

## 前端体验补充：RAGFlow 风格知识库工作台

### 设计目标

知识库模块右侧内容区可以参考 RAGFlow 的视觉和交互方式，但不直接嵌入 RAGFlow 页面，也不复制 RAGFlow 前端源码。目标是在 Soybean Admin 的后台框架内，做一个企业平台自己的 RAG 工作台。

核心原则：

- 左侧仍保留 Soybean Admin 的企业后台导航。
- 右侧知识库模块采用 RAGFlow 风格的轻量工作台布局。
- 页面数据全部来自自建平台 API，由 NestJS 再调用 RAGFlow API。
- 权限、组织、角色、审计全部由自建平台控制。

### 页面结构

进入“知识库”菜单后，右侧内容区采用如下结构：

```
知识库工作台
  ├─ 顶部胶囊导航
  │   ├─ 首页
  │   ├─ 知识库
  │   ├─ 聊天
  │   ├─ 搜索
  │   ├─ 智能体
  │   ├─ 记忆
  │   └─ 文件管理
  ├─ 右上角工具区
  │   ├─ 语言
  │   ├─ 主题
  │   ├─ 帮助
  │   └─ 当前用户
  └─ 主体内容区
      ├─ 空状态创建卡片
      ├─ 知识库卡片网格
      ├─ 文档管理视图
      ├─ 检索/问答视图
      └─ 设置/授权视图
```

### 第一阶段页面范围

第一阶段只实现 RAG 知识库核心闭环，不一次性做完整 RAGFlow 前端能力。

必须实现：

- 知识库空状态：展示“尚未创建知识库”和创建入口。
- 知识库卡片列表：展示名称、描述、文档数、切片数、解析状态、更新时间。
- 新建知识库：通过弹窗调用自建平台 API 创建 RAGFlow Dataset。
- 文档管理入口：进入某个知识库后查看文档列表。
- 文档上传：支持选择文件并上传到对应知识库。
- 文档解析：支持选择文档开始解析、停止解析。
- 知识库检索：在当前知识库内发起语义检索。

暂不实现：

- RAGFlow 原生 iframe 嵌入。
- RAGFlow 全量前端页面复制。
- Agent 画布编辑器。
- Memory 独立管理页面。
- LLM Provider 复杂配置页面。
- RAGFlow 用户/team 完整管理页面。

### 视觉要求

- 右侧内容区整体留白充足，风格接近 RAGFlow 的轻量 AI 工作台。
- 顶部导航使用胶囊式分段导航，当前项使用深色背景和高亮底边。
- 空状态使用大尺寸虚线边框卡片，中心展示图标、提示文案和新增按钮。
- 有数据时优先使用卡片网格，不把首页做成传统后台表格。
- 文档管理、任务管理等高密度信息页可以使用表格，但要放在知识库详情页内。
- 支持明暗主题，跟随 Soybean Admin 主题变量。
- 移动端可以降级为横向滚动导航和单列卡片。

### 与 RAGFlow 的差异边界

| 项目 | RAGFlow 原生页面 | 自建平台知识库工作台 |
|---|---|---|
| 登录 | RAGFlow 用户体系 | 自建平台统一登录 |
| 权限 | RAGFlow Team/API Key | 自建平台 RBAC + 资源授权 |
| 数据调用 | RAGFlow 前端直连后端 | Soybean → NestJS → RAGFlow API |
| 页面实现 | RAGFlow React 前端 | Soybean / Vue 3 |
| API Key | 用户可在 RAGFlow 中生成 | 后端托管，不暴露给前端 |
| 扩展方向 | RAGFlow 自身功能 | 数字员工、Skill、第三方系统赋能 |

### 验收标准

- 用户进入知识库模块后，右侧工作区视觉接近 RAGFlow 截图中的布局。
- 知识库为空时展示创建卡片，而不是空表格。
- 创建知识库、上传文档、解析文档、检索知识库均通过自建平台 API 完成。
- 前端代码中不出现 RAGFlow API Key。
- 普通用户只能看到自己有权限访问的知识库。
- 页面刷新、接口失败、RAGFlow 不可用时有明确错误提示，不出现无限 loading。
