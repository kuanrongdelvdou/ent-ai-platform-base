## 2. 产品定位

Forge Admin 是一个基于 Vue3 + Spring Boot 3 的企业级中后台管理框架，定位不是单纯后台模板，而是“可插拔企业应用底座”。它包含认证授权、多租户、RBAC、组织用户、菜单资源、日志审计、配置中心、文件存储、消息中心、流程引擎、代码生成、AI 能力、数据资产和 AI 数据大屏等能力。

系统适合用于：

- 企业内部管理后台基础框架
- SaaS 多租户管理系统底座
- 流程审批类系统
- 数据资产与报表管理系统
- AI 驱动的低代码/代码生成平台
- 企业 AI 应用门户的基础后台

## 3. 用户角色

### 3.1 超级管理员

负责平台全局配置、租户管理、客户端管理、系统资源、权限、日志、安全配置和服务监控。

### 3.2 租户管理员

负责本租户内组织、用户、角色、菜单授权、数据权限、文件配置、消息模板、流程模型和业务配置。

### 3.3 普通用户

使用首页、个人中心、通知公告、站内消息、我的待办、已办任务、流程申请、业务表单等功能。

### 3.4 开发/实施人员

使用代码生成、AI CRUD 生成、API 配置、数据源配置、模板配置、外部接口代理、数据集配置等工具完成二开交付。

### 3.5 数据/报表人员

使用数据连接、数据集、维度、数据大屏、AI 大屏生成和报表发布能力。

## 5. 认证与安全

### 5.1 登录

系统提供用户名密码登录，演示账号为 `admin / 123456`。登录页支持现代化视觉设计，包含系统品牌、能力卖点、账号密码输入、验证码、安全校验、记住我等能力。

功能要求：

- 支持用户名、密码登录。
- 支持 RSA 密码加密传输。
- 支持图形验证码、滑块验证码、短信验证码三种模式。
- 支持登录配置动态读取。
- 支持登录失败后刷新验证码。
- 支持记住账号密码。
- 支持登录成功后拉取用户信息、权限和菜单。
- 支持 token 持久化和自动带入请求头。

### 5.2 第三方登录

系统预留社交登录能力，文档说明支持微信、钉钉、GitHub、Gitee 等多平台。

功能要求：

- 获取启用的第三方平台列表。
- 获取第三方授权 URL。
- 支持 OAuth 回调页。
- 登录成功后统一写入 token 和用户信息。

### 5.3 安全增强

系统包含 RSA/AES/SM4 加密、防重放、幂等、接口权限、在线用户管理等能力。

功能要求：

- 请求附加 `X-Timestamp`、`X-Nonce`。
- 支持密钥交换与安全会话过期处理。
- 支持接口级权限控制。
- 支持在线用户查看和会话踢出。
- 支持登录日志与操作日志审计。

## 7. 系统管理

### 7.1 租户管理

支持多租户体系。

功能要求：

- 租户列表、查询、新增、编辑、删除。
- 租户状态启停。
- 租户用户数量限制。
- 租户描述和基础信息维护。
- 租户级数据隔离。

### 7.2 用户管理

管理系统用户及其组织、角色、岗位关系。

功能要求：

- 用户分页查询。
- 新增、编辑、删除、批量删除用户。
- 启用/停用用户。
- 重置密码。
- 绑定/解绑角色。
- 绑定/解绑组织。
- 维护用户多组织关系和主组织。
- 查看用户角色和组织列表。

### 7.3 角色管理

管理角色、菜单权限、数据权限。

功能要求：

- 角色列表、查询、新增、编辑、删除。
- 分配菜单/按钮权限。
- 配置数据权限范围：全部、本租户、本组织、本组织及子组织、本人、自定义等。
- 配置自定义组织数据权限。
- 支持角色状态启停。

### 7.4 菜单/资源管理

动态菜单和权限资源配置。

功能要求：

- 支持目录、菜单、按钮、子系统等资源类型。
- 支持动态路由路径、组件路径、图标、排序、可见性、状态。
- 支持权限标识 `perms`。
- 支持外链、iframe、SSO 子系统跳转。
- 支持隐藏菜单仍注册路由。

### 7.5 组织管理

维护企业组织树。

功能要求：

- 组织树展示。
- 新增、编辑、删除组织。
- 组织类型：公司、部门、小组等。
- 组织状态启停。
- 维护负责人、电话、地址、排序。

### 7.6 岗位管理

维护组织下岗位。

功能要求：

- 岗位分页查询。
- 新增、编辑、删除岗位。
- 岗位编码、名称、类型、所属组织、状态维护。

### 7.7 区域管理

支持行政区划/区域数据。

功能要求：

- 区域树或列表展示。
- 区域编码、名称、上级区域维护。
- 为数据权限或业务区域过滤提供基础。

## 20. 非功能需求

### 20.1 权限与隔离

- 所有管理功能必须经过认证。
- 菜单、按钮、接口、数据权限分层控制。
- 多租户数据默认隔离。
- 敏感字段支持加密和脱敏。

### 20.2 可扩展性

- 采用微内核 + 插件化设计。
- 系统模块可按需引入。
- 新业务可通过代码生成或 AI CRUD 快速接入。

### 20.3 审计

- 登录、操作、消息发送、流程操作、外部 API 调用均需有日志。
- 高危操作需记录操作者和操作结果。

### 20.4 可用性

- 核心 CRUD 页面应支持分页、搜索、重置、新增、编辑、删除、批量删除。
- 长任务应有 loading、状态反馈、错误提示。
- 重要配置变更需确认。

### 20.5 安全

- 密码加密传输。
- 支持防重放。
- 支持幂等 token。
- 接口密钥不暴露给前端。
- AI 供应商 API Key 应后端存储。

## 21. 关键业务流程

### 21.1 登录流程

1. 用户进入登录页。
2. 前端读取登录配置。
3. 按配置展示图形/滑块/短信验证码。
4. 用户输入用户名、密码和验证码。
5. 前端加密密码。
6. 后端校验账号、密码、验证码、客户端信息。
7. 返回 token 和用户信息。
8. 前端拉取菜单权限。
9. 跳转首页。

### 21.2 角色授权流程

1. 管理员创建角色。
2. 配置角色基础信息。
3. 选择菜单/按钮权限。
4. 配置数据权限范围。
5. 绑定用户。
6. 用户重新登录或刷新权限后生效。

### 21.3 流程审批流程

1. 管理员创建流程分类、表单、模型。
2. 设计 BPMN。
3. 配置节点审批人、条件、操作。
4. 部署流程模型。
5. 业务用户发起申请。
6. 审批人处理待办。
7. 系统记录审批意见、流程轨迹和业务状态。

### 21.4 AI CRUD 生成流程

1. 用户输入业务描述。
2. AI 生成元信息、表格、表单和接口配置。
3. 用户在向导中调整配置。
4. 系统生成动态页面配置或代码包。
5. 可挂载到菜单。
6. 用户访问生成后的 CRUD 页面。

### 21.5 AI 大屏生成流程

1. 用户进入大屏项目。
2. 输入自然语言需求。
3. AI 生成组件、布局、主题、数据结构。
4. 用户一键应用到画布。
5. 用户拖拽调整并配置真实数据源。
6. 保存并发布。
7. 生成预览链接。

## 22. 对企业 AI 应用平台的借鉴价值

如果用于建设“RAGFlow + 自研门户”的企业 AI 平台，Forge Admin 可重点借鉴：

- 多租户、组织、角色、菜单、按钮权限。
- 动态路由和菜单体系。
- 文件管理与存储配置。
- 消息中心和通知公告。
- 操作日志、登录日志、在线用户。
- 流程引擎和待办中心。
- AI 供应商、模型、Agent、会话管理。
- 数据连接、数据集、维度、数据资产演进方案。
- AI CRUD 生成器和页面模板机制。

不建议直接照搬的部分：

- UI 风格仍偏传统中后台，若建设 AI 平台建议重新设计 AI 工作台。
- AI 能力偏代码生成/大屏生成，不是 RAG 知识库平台。
- RAG、知识库、引用溯源、文档解析能力需要另接 RAGFlow。

## 23. MVP 建议

若基于该系统思路建设企业 AI 平台，建议 MVP 范围：

1. 登录、用户、组织、角色、菜单权限。
2. 知识库管理、文档上传、文件存储。
3. RAGFlow API 代理与问答日志。
4. AI 供应商和模型配置。
5. AI 工作台：对话、引用来源、文档预览。
6. 操作日志、登录日志、问答审计。
7. 消息通知和公告。

二期再建设：

- 流程审批。
- 数据资产。
- AI CRUD。
- AI 报表大屏。
- 外部系统 API 代理。
