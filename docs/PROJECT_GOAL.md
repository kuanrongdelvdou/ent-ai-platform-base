# 企业级 Agent 平台 — 项目目标

## 一句话描述

为企业打造一套现代化的 AI Agent 平台，融合传统后台管理系统的完整能力，以智能体驱动企业公文处理、知识管理和协作工作流。

---

## 核心功能目标

### 1. 知识库

- 支持至少 10,000 个文件的上传和管理
- 支持格式：PDF、Word、PPT、Excel、图片、纯文本等
- 文档解析、向量化、语义检索
- 知识库按部门/项目/权限隔离

### 2. 智能体（Agent）

- 公文关键内容提取
- 公文起草与写作辅助
- 文案生成
- 可配置的多步骤工作流（Workflow）
- 智能体可作为"数字员工"挂载到组织架构中（参考 NocoBase AI 数字员工概念）
- 不限于以上，支持自定义 Agent 能力扩展

### 3. 传统后台管理系统功能（完整）

- 用户管理（创建、编辑、停用、角色分配）
- 权限管理（RBAC，细粒度到功能/数据级别）
- 组织管理（部门、层级、人员归属）
- 角色管理
- 菜单管理
- 操作日志 / 审计日志（等保合规）
- 系统配置

### 4. 工作流（Workflow）

- 可视化编排
- 支持条件分支、并行、循环
- 与 Agent、知识库深度集成
- 支持审批流（公文流转场景）

### 5. 协作

- 公文协作编辑
- 版本管理
- 评论与批注

---

## 设计原则

### 不重复造轮子
- 优先使用成熟开源项目
- 只有在开源项目无法满足需求时才自研

### 参考标杆
- **NocoBase AI**：Agent 作为数字员工、权限配置、低代码扩展能力
- 吸收其设计理念，但不直接依赖 NocoBase

### 现代化 UI
- 前端底座必须是现代化设计风格
- 明确排除：若依、芋道等传统后台风格（丑、落后）
- 目标风格：简洁、现代、专业，类似 Linear、Vercel、Notion 的设计感
- 使用开源前端底座项目，减少 AI 生成 UI 时的细节描述成本

### 技术兼容性
- 技术栈之间相互兼容
- 性能不能成为瓶颈（并发几百用户）
- 本地服务器部署

---

## 技术选型（已确认）

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | Ant Design Pro (React + TypeScript + Vite) | 现代化 UI，ProTable/ProForm 开箱即用 |
| 门户后端 | Node.js + NestJS (TypeScript) | 用户/权限/组织/审计，代理调用 RAGFlow API |
| RAG + Agent 引擎 | RAGFlow | 知识库、文档解析、向量检索、Agent 创建与执行 |
| 数据库 | PostgreSQL | 用户、权限、组织、审计日志 |
| 缓存 | Redis | 会话、权限缓存 |
| 向量库 | RAGFlow 内置（Milvus/Qdrant） | 随 RAGFlow 部署 |
| 部署 | Docker + 本地服务器 | 企业私有化部署 |

### 架构职责划分

```
RAGFlow（执行引擎）
  ├── 知识库：文档上传、解析、向量化、语义检索
  └── Agent：Agent 创建、配置、执行、Workflow 编排

NestJS（门户层）
  ├── 用户管理、角色管理、组织管理
  ├── RBAC 权限控制
  ├── 审计日志（等保合规）
  └── 代理 RAGFlow API（鉴权、限流、日志）

Soybean Admin（前端）
  └── 统一 UI 入口，调用 NestJS API（已更换为 Ant Design Pro）
```

## 合规要求

- 等保合规
- 审计日志（所有操作可追溯）
- 数据加密传输（HTTPS）
- 权限最小化原则

- 文件规模：≥ 10,000 个
