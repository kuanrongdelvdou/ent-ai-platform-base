# 工作流规划

## 目标

可视化编排多步骤业务流程，支持审批流（公文流转），与 Agent、知识库深度集成。

---

## 功能模块

### 1. 工作流设计器
- 可视化拖拽编排（节点 + 连线）
- 节点类型：
  - 开始/结束节点
  - 审批节点（指定审批人/角色）
  - Agent 节点（调用 AI Agent 处理）
  - 条件分支节点（根据字段值走不同分支）
  - 并行节点
  - 通知节点（发送消息）

### 2. 工作流实例
- 发起流程（提交表单数据）
- 查看流程进度（流转图高亮当前节点）
- 审批操作（同意/拒绝/转交）
- 流程历史记录

### 3. 公文审批流
- 公文发起 → 部门负责人审批 → 主管领导审批 → 归档
- 每个节点可配置审批人规则

### 4. Agent 节点
- 工作流中的某个节点由 Agent 自动处理
- 例如：Agent 自动提取公文关键信息，填入下一个节点的表单

---

## 引擎选型

| 选项 | 说明 |
|------|------|
| RAGFlow 内置 Workflow | 适合 AI 流程编排，审批流能力弱 |
| 自研（NestJS + 状态机） | 完全自控，开发量大 |
| 开源审批流引擎接入 | 待评估（如 Activiti.js 等） |

**Phase 4 开始时再做最终选型决策。**

## 完成标准

- [ ] 可视化工作流设计器可用

- [ ] Agent 节点可嵌入工作流
- [ ] 流程实例有完整的流转记录
